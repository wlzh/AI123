OpenAI 发布 GPTs 功能已经一个月了，你尝试创建 GPTs 了吗？

####02

####关于GPTs的讨论

####GPTs可以拿来做什么？

####吴江：要更多在用户视角思考 GPTs 与场景是否匹配

今天的很多项目，大家能把想法想到，并且做到产品中，也都是一种尝试了。但还没有亮眼到让我觉得可以特别去用一下。

可能在功能上大家都做了，但是怎么样站在用户视角去解决他的问题，是不是一定要在这个场景下解决？我觉得还需要花一些时间和精力去研究场景，至少尝试站在用户视角 ，而不是单纯要把这件事通过 GPTs 的逻辑来实现。如果这个需求成立，是不是一定要在这个场景里完成？是不是一定要用这套工具和逻辑来做？ 这是一个需要思考的问题。

第二，一定要给自己的 GPTs 做 prompt 保护 ，如果没有保护的话，相当于完全没有壁垒，别人很快就可以复制你的GPTs，而且不只是思维逻辑上的防护，产品体系、策略上都需要有一定的保护思维。

####张路宇：GPTs 是创业项目快速验证 MVP 的窗口

GPTs 的技术栈相对比较简单，一方面是有不少缺陷，另外一方面是没有办法帮助创作者商业化，产品保护比较难，一个新的 idea，可能一周内就会被快速 copy，出来 100 个一样的。

所以我觉得目前的 GPTs 技术更多是给自己的想法做快速验证，验证自己早期 startup 项目的窗口。 比如说把项目做出来，快速发给认识的人，让他们感知用户场景是不是真实的，这比找一堆工程师走完整的产品验证要快得多，可以快速验证MVP（最小可行性产品） ，比如数据怎么样，是不是值得去做更多的投入。

这次演示中的 All Video X-ray 项目，背后用到了很多 api。今天的 GPTs 基于 Assistant api 可以提供很多能力，调用 api、读取知识库、以及一些复杂的 prompt 等，怎么去做一个 GPTs 综合运用到这些技术，达到一个相对好的效果，其实是有门槛的。另外，GPTs 可能只是你的产品的一个入口，还会有其他的入口帮助用户去做统计、分析、量化以及标记等。

感觉 GPTs 比较适合自媒体机构，比如说手机评测机构，基于自身的知识库搭建一个 GPTs，让用户可以跟助手进行自然语言对话，把用户流量吸引过来。

GPTs 类的产品，其实我们可以参考海外产品的做法，比如 Poe，他们也在做类似 GPTs 这样的 bot，但是 bot 后有两种不同的服务，一种是类似 GPTs 这种简单的，另外一种就是会开放服务端，开发者可以开发一些复杂的程序，前端用 bot 的形式跟用户做交互，我认为这是未来可能发生的事情。

####王俊煜：GPTs 的开发者本质上是「创作者」，GPTs 没有做到以****前做不了的事

GPTs 做出亮点还是挺难的，因为它的交互是限定的 。用了那么多GPTs，最让人 Wow 的，其实是 OpenAI 官方的 GPT Builder，花几分钟就可以 build 一个 GPT。而GPTs 本身因为交互的限制，有点像是内容的竞争 。之前看到一个分析，我也蛮认同的，对于 OpenAI 来说，GPTs 的开发者不是 App Store 的开发者，而是 YouTube 或者TikTok 的创作者 ，包括设计的商业模式也是类似的。

在交付模式给定的情况下，需要考虑的就是有什么样的东西是用自然语言做交互更合适？ 之前可能没人想到，但现在想到了，不过这个的门槛还是太低了。但是的确，用 GPTs 验证一件事情的成本低了很多。

我自己使用比较多的是一些帮助写东西的 GPTs，和常见的 prompt 不同的是，我想做的 prompt 是让它来问我问题，不断提问我，类似编辑的角色。这种其实每次用 prompt 也可以，只是因为经常用，所以 GPTs 就会方便一些。

反过来说，如果真的创业的话，也不会放到 GPTs 上，因为没法从 OpenAI 收钱，也没法向用户收钱。而且用户想体验的话，还得先付费成为 ChatGPT 的会员，太麻烦。

从产品交互来讲，GPTs 简单点说有点像导航站，其实就是一个聚合，就跟 4399 上玩小游戏一样。GPT 的很大一部分能力是包在了 action 里面， OpenAI 很重视 agent，GPTs 里我觉得能做出亮点的也是action部分，但这个需要编程能力，又需要自己有一些后端服务，门槛会高一些。

整体上我觉得 GPTs 没有去实现一些以前做不到的事情，都是以前可以做到的事情，跟之前不同的是，第一是可以很方便做到，成本可能下降了 2 - 3 个数量级，这会带来整个生态的极大繁荣。第二，也会有点流量加持，但是比较弱，因为 GPTs 只有 ChatGPT Plus 用户才能体验到。

####方汉：现在的 GPTs 非常简陋，OpenAI 还没做好

OpenAI 内部是非常重视 agent 的，这个在很多采访中都可以看到。因为大模型本身的能力有限，必须通过 agent 来落实到具体的产品上，我认为现在这个方向是没有变的。

GPTs 本身是非常简陋的 ，OpenAI都不提供付费渠道或者代码保护，很难相信这是一个顶级公司做出来的产品，所以说GPTs 还是一个实验性的产品 。我把 GPTs 的分类都抓取了一遍，排第一名的分类是 Assistant，助手类别，今天参赛的很多作品也是这个类别的，能让 GPTs 的能力更专业化。第二名是 Hobby，这个是适合传播类的 GPTs，比如最近流行的爹味语言鉴赏器，这也是最近 OpenAI 流量暴涨的原因之一。

从长远来看，要成为类似 App Store 的产品，可以回溯下苹果是怎么做的。App Store 首先把很多权限收回来，现在 App Store 仍然是最安全的一个商店，当然它也有自身很简陋的一些地方。

其次是我认为OpenAI 现在仍然没有从小白用户的角度去倒推它的产品，prompt其实是一个门槛过高的事情，虽然是自然语言交互。互联网用户花了 30 年的时间，仍然有 99% 的用户用不好搜索引擎。 埃隆·马斯克为什么对微信小程序这么欣赏，我觉得是有一定道理的，小程序才是一个更加完善的产品。

####高成志：AI 游戏可能不是游戏，不要用 AI 复刻游戏

作为游戏从业者，也看了很多AI游戏，说一说我的思考。先说一个有点反常识的尝试，AI 游戏可能不是游戏，不要想着用 AI 去复刻一个已经验证过的游戏 。如果玩家玩过这个游戏，会大概率觉得为什么不去玩原来那个游戏。用这个思维想问题，所有消费品都会说是新旧体验对比，只要是不如原来的，就会很尴尬。

第二，如果你的 GPTs 是一个泛娱乐产品，有一个核心指标，就是如果你的 GPTs 只用一张截图来做介绍，会用什么截图。我希望这张截图有非常强的社交传播力，用户看到后愿意分享，愿意打开体验。 如果产品本身链路太长，安利和做宣发是非常困难的。

既然大家是泛娱乐的精神需求，刚才也有人聊到情绪价值，这就应该把这件事发挥到极致，可以让玩家很快就能看到，而不是需要等待才能体验到。

还有就是预期管理 ，就是选一些用户没有预期或者预期很低的场景 ，现在玩 GPTs 的用户都不是泛泛之辈，一般都会带着批判性的眼光来看待产品。如果是玩家没见过的场景，比如今天展示的「旅行大象」或者「都市职场游戏」，之前没有类似的严肃游戏来承接，玩家比较陌生，就会觉得有意思，就不会对细节太严苛。

现在很多人都在用 GPTs 做 AVG、文字游戏等，回归到 AI 游戏的话题，我觉得没有必要去强调你的游戏的故事性多么完整，可以去强调短叙事的名场面，没有必要去磕长期记忆。为什么不选择一些名场面，大家觉得有趣，又有助于社交传播？而且短叙事选择要大于努力，很有可能别人想不到，你就会爆，爆了一定会有雪球效应。

####GPTs MUD 游戏是否一定要有图片？

（作为泛娱乐赛道的「参赛选手」，诺亦腾联合创始人、CTO 戴若犁开发了一款 AI 版「权力的游戏」开放式 MUD 游戏，Demo 展示时让不少观众、评委和选手惊呼暴露年龄。Demo 环节之后，Game Trigger 投资副总裁高成志与戴若犁就 AI 版 MUD 展开了讨论。）

高成志： 我提一个不太一样的观点，就是不是一定要有生图的功能，这一点我觉得要想好。比如对于塔罗游戏来说，有趣的点在哪里？塔罗就是靠有点模糊的图像给用户编造叙事，这个是有价值的，因为图片承载了信息，而且有非常多的可解读空间。但是一般的游戏，是值得商榷的。比如说游戏中生成了美女的图片，虽然很好看，但是不在我的审美区域，那我可能就会觉得这个图有点不及预期。

至于 MUD，我觉得 MUD 最大的一个优势就是没有图片，就跟读武侠小说一样，文字描写带来了更多想象，而且国人的叙事也是擅长留白的。如果场景生成的图片质量没有那么高，其实也不一定要生成图，而且在实际的体验中，图片生成的速度也很慢，会影响体验。

戴若犁： 是不是有图就扼杀了想象力，我可能需要实际再试一试。到底是把图关了更能有吸引力，还是有图更有吸引力？这个 MUD 一开始是没有图的，后来加了图之后，我初步的感觉是更有意思了一些，不过 MUD 毕竟是 20 多年前的东西了，权当怀旧。

高成志： 如果从产品的角度来看，可能就要看用户是谁。如果主打怀旧的概念，复刻了 20 年前的侠客行游戏，那把游戏的名场面或者地图还原出来，是需要有图的。如果是因为《权力的游戏》这部剧，满足 IP 粉的需求，那把名场面或者经典人物还原出来也是需要的，图片会很有说服力。

如果针对的是普通玩家，他可能会觉得 MUD 太古老了，可能我们就需要做一些硬核的游戏开发，比如职业养成或者增加自由度等，但这个又会陷入AI游戏的逻辑中，你的竞品中有更多的更牛逼的 RPG 游戏，哪怕是主打复古的也有不少。

戴若犁： 传统 MUD 是一个比较封闭的游戏，规则和故事线都是封闭的。我一开始的想法是用 GPTs 做一个 DND 规则游戏，但反倒带来的是一个开放式游戏，因为故事线是无限往外拓展的。这还挺有意思的，其实并不是完全去复原以前的 MUD，而是有了 MUD 这样的传统游戏的约束之后，更容易让玩家学会怎么玩，但是带给玩家的感受反倒非常接近一个开放游戏的感觉。
